# mortgage-calc — контекст для Claude

> **Правило:** при любом изменении логики расчётов, параметров или UI — обновить этот файл.

## Суть проекта
Калькулятор сравнения двух финансовых стратегий для покупки трёшки:
- **Сценарий А (Ипотека)**: купить новую квартиру со скидкой прямо сейчас, взяв льготный кредит (6%) + при нехватке — рыночный (18%). Старую квартиру продать через t1 лет, погасить дорогой кредит.
- **Сценарий Б (Вклад)**: деньги оставить на вкладе, старую квартиру держать, каждый месяц откладывать сбережения.

Пользователь двигает ползунки, всё пересчитывается в реальном времени.

## Ключевые функции calc.js

| Функция | Что делает |
|---|---|
| `fv(pv, rate, periods)` | Будущая стоимость PV через n периодов |
| `fvAnnuityMonthly(pmt, r, years)` | FV ежемесячных взносов pmt под ставку r за years лет |
| `calcMIRR(cf, reinvest, finance)` | Modified IRR — решает проблему нескольких знаковых переходов |
| `calculate(v)` | Главная функция: принимает объект параметров v, возвращает все метрики |
| `computeSensNPV(overrides, base)` | NPV при изменённом одном параметре (для торнадо) |

## Объект параметров v
```js
{
  p0: 50,           // рыночная цена новой квартиры, млн
  discount: 20,     // скидка при покупке, %
  equity: 15,       // свои деньги, млн
  l1: 12,           // льготный кредит, млн
  repair: 0,        // ремонт, млн
  g_new: 0.05,      // рост цен новой квартиры, годовой (decimal); ползунок 0–100%
  g_old: 0.05,      // рост цен старой квартиры, годовой (decimal); ползунок 0–100%
  r: 0.14,          // ставка вклада, годовая (decimal)
  i1: 0.06,         // ставка льготного кредита (фиксировано, не меняется ползунком)
  i2: 0.18,         // ставка дорогого кредита (decimal)
  t1: 2,            // через сколько лет продать старую квартиру (min 1)
  T: 5,             // горизонт планирования, лет
  s0: 35,           // текущая цена старой квартиры, млн
  savingsMonthly: 0.4 // ежемесячные сбережения, млн (400 тыс = 0.4)
}
```

## Логика calculate(v) — пошагово

### Шаг 1: Параметры покупки
```
C    = p0 × (1 − discount/100)    // цена со скидкой
need = C + repair                  // сколько нужно денег

если equity + l1 ≥ need:
    L2 = 0
    F0 = equity + l1 − need       // остаток своих → на вклад
иначе:
    L2 = need − equity − l1       // дополнительный дорогой кредит
    F0 = 0

I1 = l1 × i1                      // годовые % по льготному кредиту
I2 = L2 × i2                      // годовые % по дорогому кредиту
```

### Шаг 2: dealCapital(t) — капитал сценария А в момент t

**При t = 0:**
```
dealCapital(0) = p0 + s0 − l1 − L2 + F0
```

**При t > 0:**
```
cap = fv(p0, g, t) − l1           // новая квартира выросла, льготный долг висит до T

// Старая квартира:
если t ≤ t1:
    cap += fv(s0, g, t) − L2      // ещё не продали, L2 висит
иначе (t > t1):
    S1    = fv(s0, g, t1)         // цена продажи старой в момент t1
    delta = S1 − L2               // выручка после погашения дорогого долга
    cap  += delta × (1+r)^(t−t1)  // если delta ≥ 0: остаток на вклад
             ИЛИ
             delta × (1+i2)^(t−t1)// если delta < 0: нехватка растёт по дорогой ставке

// Накопленные проценты (будущая стоимость уплаченных % на дату T):
за k = 1..min(t,t1): платим (I1+I2) × (1+r)^(t−k)
за k = t1+1..t:      платим I1 × (1+r)^(t−k)

cap -= sum(paid)

если F0 > 0: cap += fv(F0, r, t)  // остаток equity на вкладе
```

**Ключевые допущения dealCapital:**
- Льготный кредит l1 гасится только в конце (год T), не при продаже старой квартиры в t1
- Дорогой кредит L2 гасится при продаже старой квартиры (год t1)
- Сбережения savingsMonthly НЕ учитываются в сценарии А (только в Б) — это известное упрощение, планируется изменить
- Долг bullet: в течение срока платятся только проценты

### Шаг 3: baseCapital(t) — капитал сценария Б в момент t
```
при t = 0: baseCapital(0) = equity + s0

при t > 0:
    baseCapital(t) = fv(equity, r, t)
                   + fv(s0, g_old, t)
                   + fvAnnuityMonthly(savingsMonthly, r, t)
```

### Шаг 4: Итоговые метрики
```
WA         = dealCapital(T)
WB         = baseCapital(T)
diff       = WA − WB
npvDirect  = diff / (1+r)^T         // в сегодняшних деньгах

totalPercent = I1×T + I2×t1         // суммарные % банку за весь срок
monthlyPay   = L2 × i2 / 12         // ежемесячный % по дорогому кредиту

// MIRR на разностных потоках (Сделка − База):
cf[0] = −(equity − F0)
cf[t] = dealCF[t] − baseCF[t]
mirrVal = calcMIRR(cf, r, r)        // MIRR > r ↔ NPV > 0 (гарантировано)
```

## Результат calculate(v)
```js
{
  C,              // цена покупки со скидкой
  L2,             // сумма дорогого кредита (0 если не нужен)
  F0,             // остаток своих денег на вкладе
  I1, I2,         // годовые проценты льготного и дорогого кредита
  WA,             // итоговый капитал по сделке
  WB,             // итоговый капитал по базе
  diff,           // WA - WB
  npvDirect,      // diff / (1+r)^T  — в сегодняшних деньгах
  mirrVal,        // MIRR в decimal; MIRR > r ↔ NPV > 0 (гарантия)
  totalPercent,   // суммарные проценты банку за T лет
  monthlyPay,     // ежемесячный % по дорогому кредиту
  saveFV,         // FV сбережений за T лет (для базы)
  baseCapital(t), // функция: капитал базы в момент t
  dealCapital(t), // функция: капитал сделки в момент t
}
```

## Главные метрики и их смысл

**NPV** = (W_A - W_B) / (1+r)^T
- > 0: ипотека выгоднее вклада в сегодняшних деньгах
- < 0: вклад выгоднее

**MIRR** — модифицированная IRR на разностных потоках с reinvest=finance=r
- > r: сделка лучше (согласовано с NPV > 0)
- < r: вклад лучше

**Торнадо** — sensitivity: каждый параметр изменяется ±delta, считается диапазон NPV.

## Допущения модели
- Долг bullet (только проценты, тело гасится при продаже/конце срока)
- Льготный кредит l1 не гасится досрочно при продаже старой квартиры
- Сбережения savingsMonthly учтены только в базовом сценарии Б
- Ставки постоянны на всём горизонте
- Налоги и комиссии не учтены
- Рост недвижимости раздельный: g_new для новой, g_old для старой квартиры
- Сбережения — ежемесячный аннуитет (fvAnnuityMonthly)
