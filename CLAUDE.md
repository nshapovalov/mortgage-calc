# mortgage-calc — контекст для Claude

## Суть проекта
Калькулятор сравнения двух финансовых стратегий для покупки трёшки:
- **Сценарий А (Ипотека)**: купить новую квартиру со скидкой прямо сейчас, взяв льготный кредит (6%) + при нехватке — рыночный (18%). Старую квартиру продать через t1 лет, погасить дорогой кредит.
- **Сценарий Б (Вклад)**: деньги оставить на вкладе, старую квартиру держать, каждый месяц откладывать сбережения.

Пользователь двигает ползунки, всё пересчитывается в реальном времени.

## Ключевые функции calc.js

| Функция | Что делает |
|---|---|
| `fv(pv, rate, periods)` | Будущая стоимость PV через n периодов |
| `fvAnnuityMonthly(pmt, r, years)` | FV ежемесячных взносов pmt под ставку r за years лет |
| `calcMIRR(cf, reinvest, finance)` | Modified IRR — решает проблему нескольких знаковых переходов |
| `calculate(v)` | Главная функция: принимает объект параметров v, возвращает все метрики |
| `computeSensNPV(overrides, base)` | NPV при изменённом одном параметре (для торнадо) |

## Объект параметров v
```js
{
  p0: 50,           // рыночная цена новой квартиры, млн
  discount: 20,     // скидка при покупке, %
  equity: 15,       // свои деньги, млн
  l1: 12,           // льготный кредит, млн
  repair: 0,        // ремонт, млн
  g: 0.05,          // рост цен недвижимости, годовой (decimal)
  r: 0.14,          // ставка вклада, годовая (decimal)
  i1: 0.06,         // ставка льготного кредита (фиксировано)
  i2: 0.18,         // ставка дорогого кредита (decimal)
  t1: 2,            // через сколько лет продать старую квартиру
  T: 5,             // горизонт планирования, лет
  s0: 35,           // текущая цена старой квартиры, млн
  savingsMonthly: 0.4 // ежемесячные сбережения, млн (400 тыс = 0.4)
}
```

## Результат calculate(v)
```js
{
  C,              // цена покупки со скидкой
  L2,             // сумма дорогого кредита (0 если не нужен)
  F0,             // остаток своих денег на вкладе
  I1, I2,         // годовые проценты льготного и дорогого кредита
  WA,             // итоговый капитал по сделке
  WB,             // итоговый капитал по базе
  diff,           // WA - WB
  npvDirect,      // diff / (1+r)^T  — в сегодняшних деньгах
  mirrVal,        // MIRR в decimal; MIRR > r ↔ NPV > 0 (гарантия)
  totalPercent,   // суммарные проценты банку за T лет
  monthlyPay,     // ежемесячный % по дорогому кредиту
  saveFV,         // FV сбережений за T лет (для базы)
  baseCapital(t), // функция: капитал базы в момент t
  dealCapital(t), // функция: капитал сделки в момент t
}
```

## Главные метрики и их смысл

**NPV** = (W_A - W_B) / (1+r)^T
- > 0: ипотека выгоднее вклада в сегодняшних деньгах
- < 0: вклад выгоднее

**MIRR** — модифицированная IRR на разностных потоках с reinvest=finance=r
- > r: сделка лучше (согласовано с NPV > 0)
- < r: вклад лучше

**Торнадо** — sensitivity: каждый параметр изменяется ±20% (или ±delta), считается диапазон NPV.

## Допущения модели
- Долг bullet (только проценты, тело гасится при продаже)
- Ставки постоянны на всём горизонте
- Налоги и комиссии не учтены
- Рост недвижимости g одинаков для обеих квартир
- Сбережения — ежемесячный аннуитет (fvAnnuityMonthly)
